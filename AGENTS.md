# 文件写入操作安全规则（快速参考）

> ⚠️ **重要**：本文件为快速参考索引，完整规范请查阅 [CLAUDE.md](CLAUDE.md)

## 📋 核心规则摘要

### 四大规则体系

| 规则类别 | 核心要求 |
|---------|---------|
| **指令约束** | 单文件串行、参数完整、JSON 校验、先代码后指令 |
| **操作执行** | 小步快跑（≤500 行）、上下文控制、手动兜底、中断异常 |
| **环境配置** | 关闭批量操作、检查中转 API、路径合规 |
| **校验验证** | 代码预校验、指令预校验、权限前置检查 |

### 核心执行优先级

1. 先遵守「指令约束规则」→ 强制 AI 输出合规内容
2. 再执行「操作执行规则」→ 避免截断/异常
3. 最后通过「校验验证规则」→ 确保写入成功

## ✅ 文件写入 Checklist

每次写入文件前必须逐项核对：

- [ ] 单文件操作，未并行写入多个文件
- [ ] `filePath` 为绝对路径字符串，非空
- [ ] `content` 为完整代码字符串，非空
- [ ] JSON 语法已自检，`{}`/`[]` 配对完整
- [ ] 代码量 ≤ 500 行，或已按模块拆分
- [ ] 文件路径无特殊字符
- [ ] Python 代码已通过 `py_compile` 语法检查
- [ ] 目标目录权限正确（`www:www`）

## 🚨 快速故障排除

| 问题现象 | 处理方案 |
|---------|---------|
| `Preparing write...` 卡住 | 按 `esc` 终止，检查 JSON 完整性 |
| JSON 解析失败 | 检查 `{}`/`[]` 配对，确认无截断 |
| 参数错误/undefined | 重新生成指令，确保参数完整 |
| 写入后权限异常 | 检查目录权限 `chown -R www:www` |
| API 响应截断 | 检查中转 API `response_max_size` 设置 |
