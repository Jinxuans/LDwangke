# 宝塔面板部署教程

> 本教程基于宝塔面板 Linux 版，部署整套系统：Go 后端 + Vue 前端 + PHP 插件模块

---

## 一、环境准备

### 1.1 宝塔安装软件

在宝塔「软件商店」中安装以下组件：

- **Nginx**（≥1.20）
- **MySQL**（≥5.7，推荐 8.0）
- **Redis**（≥6.0）
- **PHP**（≥7.4，用于运动/实习/论文插件）
- **PM2管理器**（用于托管 Go 后端进程）

### 1.2 服务器端口

| 服务 | 端口 | 说明 |
|------|------|------|
| Nginx | 80/443 | 对外，反代所有请求 |
| Go 后端 | 8080 | 内部，不对外开放 |
| PHP | 9000 | 内部（php-fpm），不对外开放 |
| MySQL | 3306 | 内部 |
| Redis | 6379 | 内部 |

> 宝塔防火墙只需放行 80 和 443 端口。

## 三、部署 Go 后端

### 3.1 编译

在开发机上交叉编译 Linux 版本：

```bash
cd go-api
set GOOS=linux
set GOARCH=amd64
go build -o go-api-linux ./cmd/server
```

### 3.2 上传文件

将以下文件上传到服务器 `/www/wwwroot/go-api/`：

```
/www/wwwroot/go-api/
├── go-api-linux          # 编译好的可执行文件
├── config/
│   └── config.yaml       # 配置文件
└── migrations/           # SQL 迁移文件（可选，执行完可删）
```

### 3.3 修改配置文件

编辑 `/www/wwwroot/go-api/config/config.yaml`：

```yaml
server:
  port: 8080
  mode: release                          # 生产环境改为 release
  php_backend: "http://127.0.0.1:9000"   # 宝塔 PHP 默认端口
  php_public_url: ""                     # 留空，Nginx 统一代理
  bridge_secret: "改成你自己的随机密钥"    # 与 auth_bridge.php 保持一致

database:
  host: 127.0.0.1
  port: 3306
  user: 你的数据库用户名
  password: "你的数据库密码"
  dbname: "7777"
  max_open_conns: 50
  max_idle_conns: 25

redis:
  host: 127.0.0.1
  port: 6379
  password: "你的Redis密码"              # 宝塔Redis如果设了密码
  db: 0

jwt:
  secret: "改成你自己的随机密钥"           # 生产环境必须修改！
  access_ttl: 7200
  refresh_ttl: 604800

cache:
  order_list_ttl: 30
  class_list_ttl: 300

smtp:
  host: "smtp.qq.com"
  port: 465
  user: "你的邮箱"
  password: "邮箱授权码"
  from_name: "系统通知"
  encryption: "ssl"
```

### 3.4 设置权限并启动

```bash
chmod +x /www/wwwroot/go-api/go-api-linux
```

#### 方式一：PM2 管理器（推荐）

宝塔 → 软件商店 → PM2管理器 → 添加项目：
- 启动文件：`/www/wwwroot/go-api/go-api-linux`
- 运行目录：`/www/wwwroot/go-api`
- 项目名称：`go-api`

#### 方式二：Systemd 服务

创建 `/etc/systemd/system/go-api.service`：

```ini
[Unit]
Description=Go API Server
After=network.target mysql.service redis.service

[Service]
Type=simple
User=www
WorkingDirectory=/www/wwwroot/go-api
ExecStart=/www/wwwroot/go-api/go-api-linux
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable go-api
systemctl start go-api
systemctl status go-api    # 确认运行正常
```

## 四、部署 PHP 旧系统 + 插件

### 4.1 创建 PHP 站点

宝塔 → 网站 → 添加站点：
- 域名：`你的域名`（或先用IP）
- 根目录：`/www/wwwroot/你的站点目录`
- PHP版本：选择已安装的 PHP 版本
- 数据库：不创建（已在第二步创建）

### 4.2 上传旧 PHP 系统

将旧 PHP 系统文件上传到站点根目录，确保以下结构存在：

```
/www/wwwroot/站点根目录/
├── confing/common.php        # 旧系统公共配置（注意拼写是 confing）
├── index/                    # 旧系统页面
├── Checkorder/               # 网课对接层
├── apisub.php                # 旧系统API路由
└── ...
```

### 4.3 上传认证桥接

```bash
# 从开发仓库复制
cp go-api/deploy/auth_bridge.php → /www/wwwroot/站点根目录/auth_bridge.php
```

**修改 `auth_bridge.php` 中的密钥**，与 Go 配置保持一致：
```php
$bridge_secret = '改成你自己的随机密钥';  // 同 config.yaml 中的 bridge_secret
```

### 4.4 部署 PHP 插件模块

运动模块（注意运动的 service/ 内容要提升一级）：
```bash
# 闪电
```

## 五、构建并部署 Vue 前端

### 5.1 修改生产环境配置

编辑 `vben-admin/apps/web-antd/.env.production`：

```env
VITE_BASE=/
VITE_GLOB_API_URL=/api
VITE_COMPRESS=gzip
VITE_PWA=false
VITE_ROUTER_HISTORY=hash
VITE_INJECT_APP_LOADING=true
VITE_ARCHIVER=true
```

> `VITE_GLOB_API_URL=/api` — 前端请求 `/api/xxx`，由 Nginx 反代到 Go 后端。

### 5.2 本地构建

```bash
cd vben-admin
pnpm install
pnpm build:antd
```

构建产物在 `apps/web-antd/dist/` 目录。

### 5.3 上传到服务器

将 `dist/` 目录内容上传到：

```
/www/wwwroot/站点根目录/admin/
```

> 放在 `/admin/` 子目录下，避免和旧 PHP 系统冲突。

## 六、Nginx 配置

宝塔 → 网站 → 你的站点 → 设置 → 配置文件，替换或添加以下配置：

```nginx
server {
    listen 80;
    listen 443 ssl;
    server_name 你的域名;

    # SSL 证书（宝塔自动生成的路径）
    # ssl_certificate    /www/server/panel/vhost/cert/你的域名/fullchain.pem;
    # ssl_certificate_key /www/server/panel/vhost/cert/你的域名/privkey.pem;

    # ============ 1. Vue 前端（管理后台） ============
    location /admin {
        alias /www/wwwroot/站点根目录/admin;
        try_files $uri $uri/ /admin/index.html;

        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
            expires 30d;
            add_header Cache-Control "public, immutable";
        }
    }

    # ============ 2. Go API 反代 ============
    # 前端请求 /api/xxx → Go 后端 /api/v1/xxx
    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/v1/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持（聊天/推送）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 86400;
    }

    # WebSocket 推送专用
    location /ws/ {
        proxy_pass http://127.0.0.1:8080/ws/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;
    }

    # ============ 3. PHP 处理（旧系统 + 插件） ============
    location / {
        root /www/wwwroot/站点根目录;
        index index.php index.html;
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        root /www/wwwroot/站点根目录;
        fastcgi_pass unix:/tmp/php-cgi-74.sock;  # 根据实际PHP版本调整
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }

    # ============ 4. 上传文件大小 ============
    client_max_body_size 50m;

    # ============ 5. 禁止访问敏感文件 ============
    location ~ /\.(git|env) {
        deny all;
    }
    location ~ /(confing|config)/ {
        deny all;
    }

    access_log /www/wwwlogs/你的域名.log;
    error_log /www/wwwlogs/你的域名.error.log;
}
```

**保存后重载 Nginx。**

## 八、验证部署

```bash
# Go 后端
curl http://127.0.0.1:8080/api/v1/site/config
# 应返回 {"code":0,"data":{...}}

# PHP
curl http://127.0.0.1:9000/index.php

# Redis
```

浏览器访问：
- `https://你的域名/admin` → Vue 管理后台登录页

### 8.3 检查 API 代理

```bash
curl https://你的域名/api/site/config
```

登录管理后台 → 运动/实习/论文 → 点击某个模块 → 应在 iframe 中加载 PHP 页面。

## 九、常见问题

### Q1：PHP 插件页面报错 "require common.php failed"
检查目录层级。运动模块（flash/pangu）的 api.php 必须在根目录下一级，因为加密代码硬编码了 `require('../confing/common.php')`。

### Q2：iframe 显示"签名验证失败"
`auth_bridge.php` 中的 `$bridge_secret` 和 Go 配置文件 `config.yaml` 中的 `bridge_secret` 不一致，改为相同值。

### Q3：Go 后端启动失败
- 检查数据库连接：`mysql -u 用户名 -p 数据库名`
- 检查 Redis 连接：`redis-cli -a 密码 ping`
- 查看日志：`journalctl -u go-api -f`（Systemd）或 PM2 日志

### Q4：前端 API 请求 404
检查 Nginx 配置中 `/api/` 的 `proxy_pass` 是否正确指向 `http://127.0.0.1:8080/api/v1/`，注意末尾斜杠。

### Q5：WebSocket 连接失败
确保 Nginx 配置中 `/ws/` 路径有 `proxy_http_version 1.1` 和 `Upgrade` 头设置。

## 十、日后添加新 PHP 插件

只需 3 步，Go/Vue 代码零改动：

1. **上传 PHP 文件**到服务器根目录对应路径（`view/index.php` + `service/api.php`）
2. **数据库插入一条记录**：
```sql
INSERT INTO qingka_dynamic_module
  (app_id, type, name, icon, api_base, view_url, status, sort, config)
VALUES
  ('新模块id', 'sport', '模块名称', 'lucide:icon名',
   '/模块名/service/api.php', '/模块名/view/index.php', 1, 排序号, '{}');
```
3. **刷新页面**，模块自动出现在对应大厅

> 详细规范见 `deploy/plugin-deploy-guide.md`
